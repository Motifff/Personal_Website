# Assistive LLM and Design Strategy: Simulating Multi-Stakeholder Roundtable Discussions (2025 研究生毕业设计)

## 项目概要
本项目是伦敦大学学院（UCL）Design for Performance and Interaction 专业的硕士毕业设计论文。
该研究探索了一种创新的用户体验范式：**基于大语言模型（LLM）的多利益相关者圆桌讨论模拟系统**。旨在帮助设计师在制定策略时，通过模拟拥有不同背景、性格和心理特征的 AI Agent（智能体）之间的辩论与协作，快速获取多元化的视角。项目核心在于构建一个动态的数字环境，并引入“意图可视化（Intent Visualization）”，让用户无需阅读海量文本即可直观追踪 AI 的行为模式与决策倾向。

## 项目主要内容

### 1. 核心痛点 (Problem)
在传统的设计调研中，收集和整合多方利益相关者（如城市规划师、环保主义者、居民等）的意见面临巨大挑战：
- **信息过载**：海量的对话文本难以处理。
- **认知负荷**：在不同角色视角间频繁切换容易导致疲劳。
- **数据碎片化**：观点散落在不同对话中，难以统筹分析。

### 2. 解决方案：Designers’ Roundtable 系统
- **OneAgent（智能体架构）**：每个 Agent 拥有独立的记忆库、性格特征和投票机制，能基于特定人设生成提案并对他人的提案进行评估。
- **RoundTable（圆桌中枢）**：作为集体智慧的中心，负责收集提案、组织投票、并根据获胜提案动态决定下一轮讨论话题，模拟真实的会议动态。
- **交互界面**：
    - **3D 圆桌会议室**：沉浸式的视觉隐喻，直观展示谁在发言。
    - **实时干预**：用户可随时暂停会议，向 Agent 注入新的记忆或强行更改讨论方向。
- **意图可视化 (Intent Visualization)**：
    - 利用 LLM 输出的 Embedding 向量和 PCA 降维技术，将抽象的观点转化为可视化的点阵。
    - **趋势追踪**：展示讨论话题如何随时间演变（例如从宽泛的“未来城市”收敛到具体的“社会公平”）。
    - **立场分析**：直观展示不同 Agent 的观点是趋同（达成共识）还是对立。

## 核心的设计思考
- **从“对话”到“社会模拟”**：超越传统的 ChatGPT 问答模式，转向基于 Agent 的社会动力学模拟（Agent-Based Modeling）。AI 不再是单一的回答者，而是具有社会属性的参与者。
- **可视化的“思维空间”**：针对 LLM 生成内容过多导致的阅读难题，提出用高维向量可视化来“看”懂讨论的走向和共识的形成，将文本阅读转化为图形识别任务。
- **人机协同的“导演模式”**：用户不再是微观的操作者，而是宏观的“导演”。设定初始条件，观察 AI 演员的互动，仅在关键时刻进行干预和引导。

## 项目内很出彩的内容
- **实验验证**：通过对比实验证明，相比于同时打开多个 ChatGPT 窗口扮演不同角色，本系统在“趋势识别”和“冲突解决”任务上显著降低了用户的认知负荷，提升了决策效率。
- **动态共识涌现**：在模拟中成功观察到了 AI Agent 之间观点的自发收敛——从最初各执一词，经过几轮辩论投票后，逐渐形成兼顾技术创新与社会公平的统一方案。
- **加载页面的微交互**：利用粒子流动的动态视觉效果作为 Intent Visualization 计算过程中的 Loading 画面，巧妙地将枯燥的等待转化为对“思维数据流”的直观感受。
